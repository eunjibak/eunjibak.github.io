13년간 브랜드 디자이너, 웹 에이전시 PM, 프리랜서, B2B SaaS 프로덕트 디자이너를 거쳤습니다. 2026년부터는 회고 AI Agent '금동이봇'을 만들면서, 한 사람을 위한 에이전트가 다른 사람의 흐름에도 닿을 수 있는지 실험하고 있어요. 변화하는 환경 속에서 발견한 인사이트를 나누고, 같은 질문을 가진 사람들과 연결되고 싶습니다.

지금 하고 있는 것

  • 고양이 회고 AI Agent 개인 행동 데이터를 분석해 감정과 행동을 인지하고 목표로 정렬을 돕는 회고 에이전트를 직접 설계해 사용하고 있어요. 메시지 캡처 → 태깅 → 상태 계산 → 회고 생성 파이프라인을 Mac Mini 서버에서 돌리고, Kimi / Claude API와 Google Calendar를 붙여 매일과 주간 회고를 받아봅니다. 지금은 같은 구조가 다른 사람의 흐름에도 닿을 수 있는지, 어떻게 상용화될 수 있을지 실험을 시작하고 있어요.
  • 디자이너의 역할 재정의 직무의 틀을 벗어나 기획부터 UX 설계, 데이터 구조, 개발까지 Claude Code로 진행했어요. 변화하는 환경 속에서 디자이너의 일이 어떻게 달라지는지 직접 만들어보며 발견한 인사이트를, 영상과 글로 정리해 같은 질문을 가진 사람들과 나누고 있습니다. 100일간 인스타그램에서 1.6만 명의 팔로워와 연결됐고, 그 흐름을 계속 이어가고 있어요.

일하는 방식

브랜드에서 프로덕트로, 다시 에이전트로. 직무가 바뀐 게 아니라, 각 단계의 지식이 다음 단계에서 연결되며 깊어졌어요.

  • 문제를 직접 듣기 에이전시 PM 시절에도, 프리랜서로 일할 때도, 사내 디자이너일 때도, 해결할 문제를 그저 해결하기보다 클라이언트나 사용자와 직접 이야기해서 진짜 문제가 뭔지부터 다시 정리했어요.
  • 구조로 풀기 보이는 화면이나 결과물보다, 그 뒤에 있는 흐름과 구조를 먼저 그렸어요. 자동화와 AI로 변화되는 지금, 작은 디테일보다 더 중요해지는 역량이라 생각합니다.
  • 반응하며 개선하기 직무의 경계가 해제되고 빠른 반영이 가능해진 지금, 기획·개발·개선을 빠른 속도로 회전하며 반영하고 있습니다.

다룰 수 있는 것

  • AI / 자동화 Claude API, Claude Code, OpenClaw, APScheduler 기반 워크플로우, Google Calendar OAuth 연동, SSH·launchctl 기반 서버 운영
  • 디자인 / 프로덕트 Figma, B2B SaaS UX, LMS·뉴스 플랫폼 경험, 문제 파악, 사용자 인터뷰 진행, 목적과 스콥에 맞는 기획과 디자인, 반영 후 개선 루프. 현재는 클로드코드로 자연어로 직접 수정하거나 자동화 워크플로우를 실험 중입니다.
  • 데이터 / 분석 행동 데이터 태깅 설계, 정책 엔진 설계, 사용자 데이터 분석 (Classum MCP 활용해 직접 데이터 확인)
  • 커뮤니케이션 비전문가에게 기술을 통역하기, 기획·개발·고객 사이에서 합의점 만들기, 팀과 현재 수준과 목표를 맞추기
